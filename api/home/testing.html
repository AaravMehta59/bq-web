API Endpoints

Our API provides a variety of endpoints for accessing and manipulating various data.

  • Uptime

    Retrieves and returns the uptime of the server.

    • Method: GET
    • Endpoint: /uptime
    • Description: Returns the uptime of the server in seconds

  • Version

    Provides the current version of the API.

    • Method: GET
    • Endpoint: /version
    • Description: Returns the version of the API

  • Get IP

    Retrieves and returns the IP address of the user.

    • Method: GET
    • Endpoint: /get-ip
    • Description: Returns the IP address of the user

  • Get Location Data

    Obtains and returns the location information of the user based on IP address.

    • Method: GET
    • Endpoint: /get-location-data
    • Description: Returns the location data of the user

  • Get Time

    Retrieves and returns the current time and date in the user's timezone.

    • Method: GET
    • Endpoint: /get-time
    • Description: Returns the current time and date in the user's timezone

  • Get Weather

    Obtains and returns the current weather status of the user based on IP address.

    • Method: GET
    • Endpoint: /get-weather
    • Description: Returns the current weather status of the user

API Usage

To use our API, simply send a GET request to the desired endpoint.

      curl -X GET http://localhost:5000/uptime

For endpoints that require additional parameters, you can specify them in the URL.

      curl -X GET http://localhost:5000/get-location-data?ip=8.8.8.8

The API will respond with JSON data containing the requested information.

      {
        "uptime": 3600
      }

API Headers

Our API uses the following headers:

  • HTTP_X_REAL_IP

    The real IP address of the user.

  • HTTP_X_FORWARDED_FOR

    The forwarded IP address of the user.

  • REMOTE_ADDR

    The remote IP address of the user.

  • Timezone

    The timezone of the user.
